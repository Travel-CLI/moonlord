Neovim

Когда дело касается Neovim - я предельно осторожен. Данный редактор может оказаться как лучшим другом, так и худшим врагом, ибо с возрастанием количества плагинов растёт и низкая производительность. Я настраивал Neovim для того чтобы работать с кодом для продакшена, в данной конфигурации нет всех фич, которые есть в IDE, однако, редактор все же становится намного мощнее.

Уже сейчас благодаря плагинам настроены и доступны возможности:

  • Переход к объявлению объекта (Go To Definition) [LSP]
  • Переход к использованию объекта (Go To References) [LSP]
  • Установка LSP прямо из редактора (Lsp install from editor) [lsp-install]
  • Дерево объектов (symbols) [SymbolsOutline]
  • Тестирование кейсов Mocha и Jest прямо из редактора (Test case from editor) [Ultest]
  • Автодополнение (Autocomplete) [cmp]
  • Качественная подсветка синтаксиса (Syntax Highlighting) [tree-sitter]
  • Интеграция с Git (Git Integration) [git-signs]
  • Быстрый поиск (Fuzzy Search) [telescope]
  • Поиска файлов в репозитории Git (Git Search) [telescope]
  • Переход к недавним файлам (Recent Files) [telescope]
  • Переход к изменениям (List Git Changes) [telescope]
  • Zen-режим (zen-mode) [ZenMode]
  • Статус-бар (Status bar) [lualine]
  • Удобный переход между буферами (Buffer swap with two hotkeys) [Bufferline]

И многое другое

Я постоянно обновляю конфигурацию и допиливаю её под свои нужды. В репозитории можно увидеть, что вкладка issues никогда не остается незамеченной, все время исправляются баги и недочеты в конфигурации, в результате данных прямых действий получается конфигурация, которая работает быстро, стабильно, а что самое главное - ей удобно пользоваться :)

Если вдруг появилось желание протестировать данную конфигурацию, то достаточно просто перенести содержимое директории neovim в $HOME/.config/nvim, а затем запустить комманду PackerSync

Tmux

Tmux - терминальный мультиплексер, который позволяет запустить сразу несколько сессий шелла в одном терминале. Я выбрал его из-за простоты использования и легкой настраиваемости. Естественно все не было так гладко при конфигурации (взять даже извечную проблему, когда цвета в Neovim, который в свою очередь запущен в сессии tmux - искажаются), однако tmux все равно остается моим любимчиком среди терминальных мультиплексеров.

Из фич уже настроены:

  • Дополнительная кнопка для действий - Ctrl + A (Screen hotkey)
  • Поддержка 256-цветов в терминале (256-color support in config)
  • Быстрая перезагрузка (Hot Reload)
  • Понятная нумерация панелей (Panel numeration fix)
  • Дополнительные хоткеи, которые увеличивают скорость взаимодействия с мультиплексером (Additional hotkeys)

Я пользуюсь данной программой каждый день, и убеждаюсь в том, что все что необходимо - уже настроено и готово к использованию другими людьми
